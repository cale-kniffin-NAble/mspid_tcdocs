: FLV recordings not uploading

Last Modified

Thu Feb 20 10:05 GMT 2020

Description

  • FLV recording files on local computer folder not uploading (C:\Users\Bruce\AppData\Local\BeAnywhere Support Express\Console\FLV).

Environment

  • /
  • Windows OS

Solution

  • Videos are temporarily stored in a local folder, that is, on the computer from which the remote session was made;
  • Videos are only uploaded to Admin Area if the console is running (no login needed) and no remote sessions are at place;
  • If the conditions above are ensured and the session video recordings are still not uploading, perform the steps below:
  1. Close your Take Control Tech Console;
  2. Navigate to the Take Control Tech Console install folder. By default is:C:\Users\Bruce\AppData\Local\BeAnywhere Support Express\Console;
  3. On the blank space inside the install folder press Shift + Right-Click and select Open Command Prompt here;
  4. Input the following command: BAConsoleApp.exe -reset_flv ;
  5. When the command finishes execution, the Logon page of the Tech Console should popup;
  6. Perform the Login on your Take Control Tech Console;
  7. Do not perform any remote sessions until the session recordings fully upload to the Admin Area.
  8. If the above command does not have the expected effect, repeat the process but use: BAConsoleApp.exe -reset_all_flvs
  • BAConsoleApp.exe -reset_flv - Resets the sending of the .flv's to our servers
  • BAConsoleApp.exe -reset_all_flvs - Resends all the pending videos, if the metadata file is present
  • If issue persists reach out to support with Take Control Technical Console logs located on your local device under C:\Users\[USER]\AppData\Local\BeAnywhere Support Express\Console\Logs.
