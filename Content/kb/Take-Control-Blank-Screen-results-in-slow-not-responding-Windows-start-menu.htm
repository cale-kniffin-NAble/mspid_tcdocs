Take Control: Blank Screen results in slow/not responding Windows start menu

Last Modified

Fri Feb 15 13:18 GMT 2019

Description

  • After enabling Blank Screen during a Take Control session, clicking the Start menu either doesn't open the menu or causes it to open slowly!
  • Search in Start Menu is slow after enabling Blank Screen during a session.

Environment

  • /;
    • Console/Agent version 7.00.00.
  • Windows.

Solution

  • Due to the nature of the Blank Screen command, some visual display drivers may be overwhelmed with the request and cause to occur slower than expected while Blank Screen is active.
  • This has been identified by our Engineering and Development team as a limitation of the current Blank Screen command that cannot, at this time, be circumvented.
  • A new Blank Screen method is expected to be developed and applied in a future update to both the Tech Console and Agent.
