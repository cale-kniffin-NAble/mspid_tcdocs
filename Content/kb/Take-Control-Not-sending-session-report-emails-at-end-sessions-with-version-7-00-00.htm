Take Control: Not sending session report emails at the end of sessions with Console version 7.00.00

Last Modified

Fri Jul 19 16:30 GMT 2019

Description

  • Setting "Send report to client when session ends" does not seem to actually send a session report e-mail;
  • Sending a Session History report from the Admin Area works;
  • Only happening with Console version 7.00.00 or prior.

Environment

  • /:
    • Windows Technician's Console version 7.00.00 or prior.

Solution

  • Due to changes in some of our domains that happened after version 7.00.00, session report e-mails sent from the Console's Viewer are no longer sent.
  • To resolve this issue, please update to our current version:
    1. Go to Take Control: Downloads.
    2. Click Windows Console to download.
    3. Run the downloaded installer.
    4. Once the install is complete, your console should be up to date and the issue should not occur.
