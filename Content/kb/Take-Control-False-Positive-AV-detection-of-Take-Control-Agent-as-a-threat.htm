Take Control: False Positive AV detection of Take Control Agent as a threat

Last Modified

Wed 13 Jan 12:09 GMT 2021

Description

  • Take Control Agent files being flagged as threats by AV software.

Environment

  • /
    • Take Control

Solution

  • Antivirus modules, such as behavioral analysis, may sometimes incorrectly flag Take Control Agent components as threats - this is usually corrected with future database updates for the AV software.
  • We are not able to prevent False Positive results in third party security software from occurring, we recommend reaching out to the support services for the AV in question and submitting a False Positive Report with a sample of the file being incorrectly marked as threat.
  • In order to prevent AV software from scanning Take Control related files and folders (and consequentially preventing false flags) we can recommend excluding the following folders and their contents, according to each product :
    • Take Control/:
      • C:\Program Files (x86)\BeAnywhere Support Express\GetSupportService;
      • C:\ProgramData\GetSupportService;
      • C:\Users\[USER]\AppData\Local\BeAnywhere Support Express\Console;
    • Take Control integrated in :
      • C:\Program Files (x86)\BeAnywhere Support Express\GetSupportService_N-Central;
      • C:\ProgramData\GetSupportService_N-Central;
      • C:\Users\[USER]\AppData\Local\MSP Anywhere for N-central;
    • Take Control integrated in RMM:
      • C:\Program Files (x86)\Take Control Agent;
      • C:\ProgramData\GetSupportService_LOGICnow;
      • C:\Users\[USER]\AppData\Local\Take Control Viewer;
