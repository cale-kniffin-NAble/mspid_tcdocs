Drawings from design software do not appear or get distorted when connecting with Take Control

Last Modified

Fri Apr 17 09:44 GMT 2020

Description

  • Contents from some design softwares are not properly displayed when connecting through Take Control.
  • This behavior has been seen with Sketchup, Autocad and Adobe Illustrator so far but other design softwares could be impacted as well.

Environment

    • Take Control
  • /

Solution

  • Certain types of surfaces are not properly detected, or detected at all, by Take Control because design softwares commonly use an hardware accelerated rendering mode that Take Control is not able to capture;
  • This happens due to the way Take Control works as it captures the image through the remote device's Display Adpater's output instead of installing it's own driver, which has its advantages but can cause some constraints, specially with design software;
  • To circumvent this it's advised, if possible, to change the rendering mode applied to these design programs and adjust it to CPU mode/level or similar (this is usually addressed on software's documentation).
