The Take Control Agent ( Service) is marked as an interactive service however the system is not configured to allow interactive services

Last Modified

Wed May 13 11:52 GMT 2020

Description

  • The System Event Log has the following error:
    • The Take Control Agent ( Service) is marked as an interactive service.
    • However the system is not configured to allow interactive services.

Environment

    • Take Control
  • /
  • Windows OS
    • versions prior to Windows 10 v.1803

Solution

  • This occurs because the Take Control service has the "Allow service to interact with desktop" option enabled;
    • This is for legacy purposes tied to Windows XP and Server 2003 devices, which are supported with standalone Take Control/
  • To disable this option, please do the following:
    1. Open up Services on the local device as Administrator;
    2. Double click the Interactive Services Detection service;
      • from Windows versions 1803 onwards this service does not exist;
    3. Stop the service and set the startup to Disabled;
    4. Click OK;
    5. Double click on the Take Control Agent () service;
    6. Click on the Log On tab;
    7. Untick Allow service to interact with desktop;
    8. Click OK;
  • For more information on interactive services, please go here:
    • https://docs.microsoft.com/en-us/windows/win32/services/interactive-services
