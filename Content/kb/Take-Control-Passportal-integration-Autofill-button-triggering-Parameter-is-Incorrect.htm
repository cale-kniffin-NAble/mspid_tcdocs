Take Control: Passportal integration Autofill button triggering "Parameter is Incorrect"

Last Modified

Fri Nov 13 12:50 GMT 2020

Description

  • Using the 's Passportal Integration's AUTOFILL button on the Windows login screen is triggering a "Parameter is Incorrect" message for Domain Users and Admins.

Environment

  • / Console
    • Viewer

Solution

  • Due to how Take Control sends credentials to the Windows API, the autofill option will only work for Domain User credentials if the Domain is actually included in the username (example: username@domain or domain\username).
  • As a work around and should changing username values in Passportal for these credentials not be an option, please consider using the Username Copy and Password Copy buttons (left of Autofill) and paste them into the respective fields.
