: Tech Console asks Take Control password while connecting with no Take Control password set on the remote Agent

Last Modified

Tue 12 Jan 13:04 GMT 2021

Description

  • When connecting to a computer from the computer list a prompt comes up requesting for a Take Control password;
  • There's no Take Control password defined in the agent;
  • The console version is 6.77.xx or earlier;
  • The agent version is 6.80.20.

Environment

  • /;
    • Console version 6.77.xx;
    • Agent version 6.80.20.
  • Windows.

Solution

  • The issue is caused by the older console versions not supporting the latest authentication protocol.
  • An automatic Tech Console upgrade is being pushed to the customers running older versions of the console (6.80.17 or earlier) which corrects the issue. The upgrade can also be done manually, as follows:
    1. Go to Take Control: Downloads.
    2. Click Windows Console.
    3. Run the downloaded installer.
    4. Once the install completes, the Tech Console should be up to date and the issue cleared.
