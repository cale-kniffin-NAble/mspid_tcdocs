Stop the Take Control Agent (Standalone) from launching a process per user on a terminal server

Last Modified

Mon Dec 23 17:34 GMT 2019

Description

  • How to stop the Take Control Agent from launching the Agent Configuration process per user on a terminal server;
  • Note that the BASupSrvcCnfg.exe process runs per user and is used by the Take Control Agent in order to, among other things, provide the remote support chat window and agent related prompts on the Agent's end.

Environment

  • /.

Solution

  • While it is not possible to prevent this process from running while keeping all Agent features intact, you can still delete the registry entry for BASupSrvcCnfg.exe to run on startup as follows:
    1. Start the Run desktop application in the remote device (Windows key+R);
    2. Input regedit and press OK (you may need to approve a UAC prompt);
    3. Right-Click the registry entry under C:\HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\Windows\CurrentVersion\Run and select Export in order to store a registry key backup;
    4. Delete the registry entry under C:\HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\Windows\CurrentVersion\Run;
      • Be aware that after these changes you will not be able to use the chat feature and the remote user will not have the ability to "Request Support" from the Agent.
      • Your ability to remotely access the device will not be affected, however.
