Take Control: Going offline permanently after an update, service has been marked for deletion on any devices running Site24x7

Last Modified

Mon Nov 12 17:07 GMT 2018

Description

  • Every time a Take Control service update occurs on a server running Site24x7, the service becomes marked for deletion and the Agent never comes back online.

Environment

  • /;
  • Windows devices with Site24x7 (RMM Software) running on them

Solution

  • The Site24x7 RMM software in use will lock services, preventing the Take Control Agent update from completing as expected and having the Take Control Agent service be stuck in a Marked for Deletion state.
  • This issue can be solved by rebooting the device and re-installing the Take Control Agent;
  • Alternatively, you can also disable the Site24x7 software and run the Agent Updater to repair the install and restart the service;
  • To prevent future automated updates from interfering with the Site24x7 software you can also disable automated updates as follows:
    1. Navigate to C:\Program Files (x86)\BeAnywhere Support Express\GetSupportService
    2. Create a text file on this location with the following text: mspa_install_lock
    3. Change file name and extension to precisely: __installing.lock
