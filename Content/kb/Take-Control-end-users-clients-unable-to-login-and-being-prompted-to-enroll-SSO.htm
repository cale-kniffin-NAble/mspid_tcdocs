Take Control end users/clients unable to login and being prompted to enroll SSO

Last Modified

Tue Oct 20 15:49 GMT 2020

Description

  • Since March 14th 2019 it's mandatory for all end users/clients to enroll on SSO (Single Sign On).
  • Previously created end users will not be able to login on the end user area unless they perform the SSO enrollment correctly.

Environment

Solution

  • The following steps must be taken to correctly enroll an end user on SSO:
  1. Make sure the end user has a valid email as a username;
    • Otherwise please have this changed in the Admin Area;
  2. Log in to the Admin Area;
  3. Navigate to Management > Technicians.
  4. Select the intended end user and click the button Send SSO invite;
  5. Instruct the end user to access their email address, open the SSO invitation email and click the Create SSO Account link;
  6. A new tab will open directing them to an SSO enrollment form;
  7. Fill the form as intended;
  • The end user can choose to keep the same password or define a new one, as well as their first and last name for the end user profile.
  • End users created from March 14th onward will automatically receive the SSO invitation on their email.
