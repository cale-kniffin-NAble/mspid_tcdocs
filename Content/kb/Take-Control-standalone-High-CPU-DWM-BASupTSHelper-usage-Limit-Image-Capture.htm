: High CPU/DWM/BASupTSHelper usage (Limit Image Capture)

Last Modified

Wed Dec 30 10:32 GMT 2020

Description

  • The Take Control Agent is using a large percentage of the CPU during remote sessions on the client's end;
  • The process dwm.exe is consuming a high percentage of CPU;
  • The process BASupTSHelper.exe is consuming a high percentage of CPU;
  • How to Limit Image Capture during Take Control or sessions?

Environment

  • /

Solution

  1. Access the remote device;
  2. Open the Take Control Agent Configuration application;
    • (Windows) You can find it on the sytem tray (bottom right corner of your desktop) or by searching for the Take Control Agent Configuration application;
  3. Click the Enter Administration Mode button;
  4. Click the Advanced Setup button (cogwheel icon);
  5. Select Settings;
  6. Mark the option Limit image capture (uses less CPU on older PCs) ;
  7. Click Apply;
  8. Start (or Restart) a session to the afflicted device with these new settings.
    • You should notice a significant decrease in CPU/DWM/BASupTSHelper usage during remote sessions.
