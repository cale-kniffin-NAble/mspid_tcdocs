: The maximum number of installations was reached or this installer expired!

Last Modified

Thu 04 Feb 23:33 GMT 2021

Description

  • Error: The maximum number of installations was reached or this installer expired.

Environment

  • /

Solution

  • One of the following issues is occurring:
    • You have reached your license limit for installations (500 per license):
      • Verify the number of agent installations you already have deployed via the Admin Area:
        • Note that per license you're entitled to either 50 (Take Control) or 500 () Agent installs - this includes Online and Offline Agents.
        • Should your Agent total exceed the limit, please remove sufficient Agents to be under the limit in order to allow for new installs.
    • You have reached the installation limit or expiration date for the Agent installer in use:
      • Generate a fresh/new Agent installer with it's expiration date set in the future. For more information, see [%=General.ProductName_tc-sa%]: How to generate an Agent Installer (Unattended Support).
    • End user system time is not synchronized properly:
      • Verify and fix, if necessary. the system time on the device to match it's timezone, allowing the Agent to check in with the relevant infrastructure and register the installation.
    • Necessary Windows components are damaged and not allowing the installer to run as intended:
      • Consider repairing Windows through an sfc /scannow command or repairing the MSXML Parser component: https://www.microsoft.com/en-us/download/confirmation.aspx?id=15697

Internal notes: Page tags:, article:problem, visibility:public, stage:approved, agents, installation, devices, limits, n-central, maximum, msp anywhere, parsers, msxml- This could also happen if the installer is launched on a device with an existing installation, and that installation is expired. This happens because the installer looks for the file "BASupSrvc.ini" on "C:\ProgramData\GetSupportService" and if this file exists, it uses the "InstallID" value there, instead of its own - if that "InstallID" value is expired, the error will still be displayed. So, if the Partner wants to reinstall the Agent and is getting this message, they should uninstall the previous Agent before, or at least delete that file.
