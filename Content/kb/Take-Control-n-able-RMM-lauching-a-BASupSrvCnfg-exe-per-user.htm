(): lauching a BASupSrvCnfg.exe per user

Last Modified

Fri Dec 20 16:01 GMT 2019

Description

  • The Agent installs the necessary services to the utilization of the application, namely the Agent Service and the Update Service.
  • Besides the described services, the Agent will have a configurator process running on the machine (BASupSrvcCnfg.exe).
  • Although the Agent is a one-off process per machine, the configurator is a single process per user.
  • That is, there is a need to include the configurator process in each guest/user session, to allow the use of the Agent configurator. This happens to allow interaction in each user session running on the server.
  • Therefore is not unusual to see several BASupSrvcCnfg.exe processes running o a machine. You'll have one to each user/guest on the server host.

Environment

Solution

  • If you do not want to have the process BASupSrvcCnfg.exe running on a machine, you can block its execution at the Start of the machine.
    1. Click Search Windows
    2. Type MSCONFIG
    3. Click System Configuration
    4. Click Startup
    5. Click Open Task Manager
    6. Click Startup
    7. Right-click Agent
    8. Click Disable
    • This way, the server/workstation being (re)started will not launch said process, however, you'll lose the possibility of using the chat, given that this function is interpreted by the configurator (the process BASupSrvcCnfg.exe).

  • You can also block the process / stop its execution and initialization, by removing the correspondent entry on the Windows Registry.
    1. Type Regedit on Search Windows
    2. Click Regedit
    3. Navigate to Computer\HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\Microsoft\Windows\CurrentVersion\Run\BASupSrvcCnfg_LOGICnow
    4. Right-click BASupSrvcCnfg_LOGICnow and choose Delete
      • In Run you can right-click and Choose Export (like this you keep a registry key backup) and then delete "BASupSrvcCnfg.exe".
