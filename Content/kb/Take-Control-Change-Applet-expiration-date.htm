Take Control: Change Applet expiration date

Last Modified

Fri Nov 09 11:16 GMT 2018

Description

  • By default, Take Control Applets are set to expired within 12 hours of being created - this falls within the design of Applets serving to start an Attended Support request from the client's end without installing a long-term program on their device;
  • How to extend or reduce your Applets' expiration date, be it in order to allow a client multiple uses of the same executable over a series of days or to ensure that they can only execute Applets within an expected time-frame?

Environment

  • .

Solution

  1. Login to your Admin Area ;
  2. Navigate to Profile > Identification
  3. Select the PIN/CALLING CARDS tab
  4. Set the Applet expire after value to your preferred setting (from 1 hour to Never Expires);
    • This setting will only apply to all new Applets created after saving the intended changes - previously generated Applets will not be changed;
    • Although available, we advise against the use of the "Never expires" setting - as it potentially allows the remote user to, at any time, repeatedly send support requests, regardless of whether or not they're merited.
  5. Save when done (bottom right corner).

Article Number: 116453

Internal notes: Remind partners that, should they choose to have Applets set to NEVER EXPIRE, that we can't then disable the Applets and that there's nothing stopping an End User from, for example, running the Applet indefinitely
