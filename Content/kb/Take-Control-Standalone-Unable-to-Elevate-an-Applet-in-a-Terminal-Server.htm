: Unable to Elevate an Applet in a Terminal Server

Last Modified

Thu Apr 18 12:14 GMT 2019

Description

  • When in an Take Control Applet Session, you try to elevate your privileges on a terminal server, inserting the Admin Credentials from you side but you're always unable to elevate the session.

Environment

  • /

Solution

  • Due to the limitations imposed by the OS running as TS. The two alternatives for this are either:
    • Disable the UAC for the TS server
    • Use the Agent instead.
