: Unable to be logged into the Admin Area and Tech Console simultaneously

Last Modified

Fri Jan 08 10:53 GMT 2021

Description

  • Getting kicked out from the Tech Console, when trying to login on the Admin Area or vice versa.

Environment

  • Plus (Standalone)

Solution

  • This occurs when the maximum number of Take Control licenses are at place, given that the number of Take Control licenses is equivalent to the number of simultaneous logins;
  • This issue can be bypassed if the Remote Control Mode is disabled on the Admin Area so a license is not being consumed;
    1. Login to your Admin Area;
    2. Click the Remote Control Mode Icon near the top right corner of the admin page (the icon should look like a bold dot with 2 curved lines to it's left);
    3. You'll get a pop up message saying "To disable this mode, you need to log out of the Admin Area. If you have your account configured to automatically log in on Remote Control Mode, you need to disable it first by going to Management > Technicians in the left navigation pane",
    4. Click Ok;
    5. Logout from the Admin Area.
    • Note that if you want to launch remote sessions from the Admin Area you'll need to keep the Remote Control Mode enabled and it's expected you'll be kicked off your technician console should you have reached the maximum number of licenses.
  • It's advisable to disable the Remote Control Mode by default:
    1. Go to your Admin Area;
    2. Navigate to Management > Technicians;
    3. Select the intended tech;
    4. Select the DEFAULT SETTINGS tab;
    5. Turn off the option Remote Control Mode on Admin Area logon;
    6. Click Save.
