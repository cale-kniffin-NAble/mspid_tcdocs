Take Control (All Versions): Differences between Closing and Disconnecting a session

Last Modified

Tue Jan 05 12:57 GMT 2021

Description

  • Why is the wallpaper not being restored after a remote session?
  • How to properly end a remote support session?
  • Can I disconnect a session temporarily and then reconnect to the same session?

Environment

  • /
    • Take Control

Solution

  • Clicking X to close the Viewer window will properly terminate the remote session by sending the quit / rest command, reverting the changes performed when starting the session, such as: graphical optimizations, changed resolution, restoring wallpaper etc;
  • Selecting the Session > Disconnect option will cause the session to disconnect but not finish, as it's possible to reconnect with all the optimizations, changed resolutions, disabled wallpaper etc prevailing until the session is properly terminated;
  • If the wallpaper is stored on a UNC path it may not be restored after properly terminating the session since the Take Control agent runs at the Local System which by default does not have access to network shares - this is Microsoft's design.
