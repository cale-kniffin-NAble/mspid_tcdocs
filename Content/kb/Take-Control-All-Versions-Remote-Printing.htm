Take Control (All Versions): Remote Printing

Last Modified

Tue May 19 13:22 GMT 2020

Description

  • How to enable and make use of remote printing on Take Control?

Environment

    • version 6.80.21 (Windows) or above.
  • :
    • Take Control Windows Viewer version 6.80.21 (Windows) or above.
  • / as of Console version 6.80.21 (Windows) or above.

Solution

  1. Launch a remote session to the intended Agent
  2. Click the Remote Desktop tab
  3. Select the Interactions menu (either from the lower right corner or from the Remote Desktop drop-down menu at the top left corner of the window)
  4. Select Enable Remote Printing (this is a per session setting)
  5. Print a file or text on the remote end
  6. On the printer select interface, select the Remote Printer and click Print
  7. A print job will begin on the local end:
    • If a local Default Printer is not set, choose the printer to use locally and click Print
    • If there is a local Default Printer, the print job is sent to that printer automatically
    • Note that you can select a Local Default Printer from the Viewer window (as of Take Control 7.00.00), from the Preferences > Local Default Printer menu (top right corner)
  • On remote Windows XP or Vista computers, an extra component might be required to support XPS: in the case of Vista, this is an extra Windows module, that needs to be added through the Add/Remove Windows features - on XP, it is an external, downloadable component from Microsoft. The user will be warned of this through the display of a message if the installation of the driver fails. As an extra information, Teamviewer has the same exact need, but according to our tests, they simply fail the install of the driver, without giving any feedback to the user.
