Take Control (All Versions): Blank Screen results in Windows lock on remote end

Last Modified

Tue 12 Jan 17:32 GMT 2021

Description

  • Attempting to use the Blank Screen option during a Take Control session locks the remote Windows device.

Environment

  • /
    • Take Control

Solution

  • This is caused by the remote device's Sign-In settings requiring a Windows Sign-In when waking from sleep (which is triggered by the attempt at sleeping the display with the blank screen option).
  • A change on the remote device's windows settings will be required:
    1. Open the Settings option (this option will usually be located in the Start menu/screen - you can also search for Settings or Users to skip step 2);
    2. Select Users (or Accounts if Windows 10);
    3. Select Change (or Sign-in Options if Windows 10);
    4. Set Require Sign-In to Never (as opposed to When PC wakes up from Sleep).
