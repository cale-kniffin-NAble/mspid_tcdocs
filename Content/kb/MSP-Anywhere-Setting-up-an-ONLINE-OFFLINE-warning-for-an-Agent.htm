: Setting up an ONLINE/OFFLINE warning for an Agent

Last Modified

Thu Nov 19 17:56 GMT 2020

Description

  • How to enable monitoring and setup an ONLINE/OFFLINE alert (with email communication sent to your desired address)?
  • Please note that these features do not replace a fully fledged RMM solution and will be monitored and communicated by the remote Agent software as opposed to a monitoring agent on a server.

Environment

Solution

  1. Login to your Admin Area;
  2. Navigate to Devices;
  3. Click on the more options > edit option to the right of the intended device;
  4. Check the Enable Monitoring option;
    • This may be unavailable due to the number of monitoring licenses allowed (up to 10) - depending on the number of licenses already in use, you may need to disable previous monitoring licenses.
  5. Save the changes on the bottom right corner and refresh the page;
  6. Select the now monitored device again;
  7. Select the ALERTS tab (south pane);
  8. Press Add Alert (top right corner);
  9. Set an Alert Name and select Device is Offline as the Rule Type;
  10. Under the name, choose Send Email - fill the text field under Send Email if selected;
    • This step is optional as you may only want the prompts to register in the Alert History.
  11. Press Save (bottom right corner of the prompt);
  12. The Agent is now monitoring the device and will prompt an email alert (assuming step 10 was performed) whenever it is able to verify or loses network access.
    • Note that a sudden OFFLINE status (not caused by a standard shutdown) may not be communicated immediately and will only be prompted when our infrastructure verifies that the device is offline (which is checked around every 5 minutes).
    • If a device goes offline and recovers in under 5 minutes, you may receive the ONLINE and OFFLINE alerts simultaneously after the device is back online.
