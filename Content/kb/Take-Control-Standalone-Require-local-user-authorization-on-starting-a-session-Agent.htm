: Require local user authorization on starting a session (Agent)

Last Modified

Fri Aug 21 08:58 GMT 2020

Description

  • How to require local user authorization on starting an Agent session;
  • How to disable local user authorization;

Environment

  • /

Solution

  • Per Agent:
    1. Connect to the remote device;
    2. Restore/Load the Take Control Agent Configuration window;
      • It should be located in the systray, but you can also use the search option to locate it by name "Take Control Agent Configuration";
    3. Press the SECURITY button;
      • If you can't see a SECURITY button, press the padlock icon to the top right of the window - once unlocked, you should be able to view it;
    4. Enable/Disable the Require local userauthorization on starting a session option;
    5. Press APPLY;
      • Enabling this option will prevent you from accessing this Agent without the previous approval of a local user - apply it only when needed/requested by the user as it may compromise future needs for Unattended Remote Support.
  • This setting is disabled by default. To enable this as the default for future agent installations proceed as follows:
    1. Login to your Admin Area;
    2. Navigate to Profile > Agent Setup;
    3. Select the SECURTY tab;
    4. Enable the option Remote sessions require local user authorization;
    5. Save when done (bottom right corner);
      • Note that these settings will only apply to NEW installations of the Agent performed after this point, existing Agents will need to have this setting changed manually.
