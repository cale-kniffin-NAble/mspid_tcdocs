Take Control: Remote Desktop stuck on Please Wait message after Remote Device enters in Power Saving mode

Last Modified

Tue Oct 23 13:48 GMT 2018

Description

  • "Please Wait" Message on Remote Desktop tab.
  • System Shell, File Transfer, General and System Info tab all work as intended.
  • Happens after remote device hibernates/enters a power saving mode.
  • Agent logs state: [Event] - System exited low-power state...

Environment

  • MacOS 10.8 (Mountain Lion) or above
  • Windows 7 or above

Solution

  • For MacOS, edit the remote device's Energy Saver preferences as follows:
    1. Navigate to Preferences>Energy Saver
    2. (Recommended setting) Set Computer Sleep to Never
    3. (Recommended setting) Set Display Sleep to Never
    4. Enable Wake for network access
      • Certain portable macOS devices will not wake their display (as it's sleep is managed separately) on network activity - however, with this option enabled, you should be able to interact with the device even with Remote Desktop unable to capture the display - for these cases, we recommend setting the Display Sleep to Never
  • For Windows 7 or above, edit the device's Power Settings as follows:
    1. Open the Control Panel
    2. Select System and Security
    3. Select Power Options
    4. Click Change Plan Settings and change the power plan as follows:
      • (Recommended Setting) Set Display Sleep to Never;
      • (Recommended Setting) Set Computer Sleep to Never;
    5. Save Changes when done
