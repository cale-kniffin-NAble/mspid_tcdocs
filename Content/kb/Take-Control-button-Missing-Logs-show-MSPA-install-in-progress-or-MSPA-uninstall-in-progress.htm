Take Control button is Missing - Logs show "There is currently an MSPA install action in progress" or "There is currently an MSPA uninstall action in progress"

Last Modified

Wed Aug 19 12:19 GMT 2020

Description

  • has been enabled
  • RBM has been enabled
  • The Take Control button is missing, and never appears
  • The following shows in the C:\Program Files (x86)\Advanced Monitoring Agent\debugfeature.log:
    • MSPAInstaller::Install Initiating MSPA agent install process
      MSPAInstaller::Install MSPA There is currently an MSPA uninstall action in progress
    • OR
    • MSPAInstaller::Install MSPA There is currently an MSPA install action in progress

Environment

  • Take Control
  • RBM Advanced

Solution

  • When the Take Control Agent installs, it puts down an installing lock file, so that the Advanced Monitoring Agent knows the Take Control Agent is still installing.
  • After a successful install, the Take Control Agent removes this file.
  • This is the same for an Uninstall action of the Take Control Agent.
  • However, if an Antivirus software terminates the Take Control Agent mid install or mid uninstall, that lock file is left behind
  • This causes the Advanced Monitoring Agent to wait indefinitely for the Take Control Agent to finish the operation it was doing before it was terminated
  • To solve this, please run the following script on the device, which will detect the lock situation, and remove all traces of Take Control, allowing Take Control to install successfully:
    • https://s3.amazonaws.com/new-swmsp-net-supportfiles/PermanentFiles/InstallingLockDetection.ps1
