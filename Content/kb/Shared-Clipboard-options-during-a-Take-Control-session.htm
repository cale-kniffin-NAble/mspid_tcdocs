Shared Clipboard options during a Take Control session

Last Modified

Wed Apr 22 16:19 GMT 2020

Description

  • When connected to a device, by default, is the clipboard shared between the local device (Technician's end) and the remote device (Client's end)?
  • How to change clipboard sharing settings?
  • How to disable clipboard sharing during Take Control sessions?

Environment

  • /
    • Technician Console Application 6.80.17 onwards;
  • :
    • 6.80.17 onwards.
    • Viewer 6.80.17 onwards.

Solution

  1. Start a remote session through Take Control;
  2. Select the Remote Desktop tab;
  3. Select the Interactions menu;
  4. Select the Shared Clipboard submenu and:
    • Set it to Disabled if you want to disable automatic synchronization between local and remote clipboards.
    • Set it to Local to Remote if you want to enable one-way synchronization from the local clipboard to the remote clipboard only.
    • Set it to Remote to Local if you want to enable one-way synchronization from the remote clipboard to the local clipboard only.
    • Set it to Bidirectional if you want to enable automatic synchronization between local and remote clipboards.
