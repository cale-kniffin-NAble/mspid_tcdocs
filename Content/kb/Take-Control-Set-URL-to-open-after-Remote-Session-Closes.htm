Take Control: Set URL to open after Remote Session Closes

Last Modified

Tue Sep 10 17:57 GMT 2019

Description

  • I have my own site for surveys that I'd like to have the end user directed to when the session ends, can I do this with Take Control?
  • After ending an Attended Access, how can I direct the client to a specific URL?

Environment

  • :
    • Applet.

Solution

  • You can setup this setting for Applets (attended support option without a permanent install) as follows:
    1. Login to the Admin Area;
    2. On the left-hand menu panel, select Profile;
    3. Select Applet Setup;
    4. Select the WARNINGS tab;
    5. Under Applet Closed, check the Enabled option (set it to green);
    6. Set the URL to open value to the intended URL;
    7. Check either:
      • Show everytime the applet is closed: The URL will be loaded into the default browser every time the Applet is closed by the client, regardless of the cause of closure.
      • Show only when the applet is prematurely closed by the user: The URL will be loaded into the default browser only when the Applet closes prematurely (force-closure, unexpected crash, etc.).
    8. Save (bottom-right corner) when done.
      • Please note that this setting will only apply to Applets (attended support executables without a permanent install) and not remote sessions to Agents (permanent unattended support installs).
      • Please note that changed Applet settings will only apply to Applets generated after the settings are saved, previously generated Applets will keep the settings they had when they were originally generated.
