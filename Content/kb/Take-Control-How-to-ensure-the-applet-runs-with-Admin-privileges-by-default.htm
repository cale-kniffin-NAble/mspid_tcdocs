Take Control: How to ensure the applet runs with Admin privileges by default

Last Modified

Wed Oct 16 11:18 GMT 2019

Description

  • Instructions on how to force applet executables run with administrator privileges by default.

Environment

  • /

Solution

  1. Login to your Admin Area;
  2. Navigate to Profile > Applet Setup;
  3. Under the SETTINGS tab, tick the option Always try to run with Administrator privileges, where applicable;
  4. Save all changes.
  • If the remote user doesn't have Admin Privileges nor Admin Credentials, the applet won't run as Admin and the session will be launched with lower privileges.
  • This behavior can't be bypassed as the applet runs within the user profile running it. i.d. it's tied with the remote user's privileges.
