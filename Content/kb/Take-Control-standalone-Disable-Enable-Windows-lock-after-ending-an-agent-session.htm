: Disable/Enable Windows lock after ending an agent session

Last Modified

Wed Jun 03 11:25 GMT 2020

Description

  • How to stop Take Control to lock the remote user's Windows session after terminating the remote session?
  • How to enable session lock after terminating a remote session?

Environment

  • /

Solution

  • This can be done directly on the remote device or through the Admin Area;
    • By directly accessing the remote device:
    1. Access the remote device;
    2. Restore/Load the Take Control Agent Configuration window;
      • It should be located in the systray, but you can also use the search option to locate it by name "Take Control Agent Configuration";
    3. Press the Advanced Setup button (cogwheel icon);
    4. Should it be grayed out, press the Enter Administration Mode button on the top right corner to enable Admin Mode;
      • You'll need Administrative Privileges to proceed so a pop up window will appear informing about this;
    5. Select Security;
    6. Check/Uncheck, as desired, the option Lock Windows when the remote session ends;
    7. Apply the intended changes;
    • Through the Admin Area:
      1. Login to the Admin Area;
      2. Navigate to the Devices menu;
      3. Select the intended device(s) from your device list;
      4. Click More > Apply Configuration;
      5. On the Target Device(s) section select Apply to selected or Apply to groups as intended;
      6. Choose the desired action for the option Lock Windows when the remote session ends according to the standard symbol meanings;
        • - = Do not change (Leave as is)
        • <blank> = Uncheck (Disable)
        • Check mark = Checked (Enable)
      7. Click Apply to deploy changes.
      8. Usually it takes up to 1 or 2 minutes to propagate these changes but this can vary according to device specs as well as the local and remote network environments at place.
