Take Control: Keystrokes don't show up on remote end or display the wrong character when ActivTrak is enabled

Last Modified

Wed Nov 13 12:16 GMT 2019

Description

  • Not able to type alphanumeric (a,b,c,1,2,3,etc) or modifier key characters (shift+1,shift+2,etc.) to the remote desktop for up to 5 minutes. The only keyboard inputs that are parsed are those of non-alphanumeric characters or modifier key characters, such as '.' ',' '?';
  • Mouse input works as intended;
  • ActivTrak is enabled on the local end/technician's end.

Environment

  • /;
  • Windows;
  • ActivTrak.

Solution

  • ActivTrak will interfere with the 's capture of local keyboard inputs. Please consider disabling ActivTrak or, if possible, excluding files under C:\Users\Bruce\AppData\LocalBeAnywhere Support Express\Console\ from ActivTrak, to avoid this interference.
  • Alternatively, please attempt to perform the following edit to the Viewer's .ini file, which changes how key presses are captured - which should avoid ActivTrak's interference:
    1. Close your Take Control Technician's Console entirely;
    2. Navigate to "C:\Users\<USERNAME>\AppData\Local\BeAnywhere Support Express\Console\";
    3. Open the "BASEClient.ini" file;
    4. Under [Main] Add the line DisableLLKeybMode=1:
      • [Main]
        DisableLLKeybMode=1
        (...)
    5. Save the file;
    6. Restart your Take Control Technician's Console.
