End user is presented with a permissions dialog box on starting Take Control session to a macOS Catalina host

Last Modified

Wed Jul 15 16:49 GMT 2020

Description

  • I am unable to start a Take Control session to a macOS Catalina host
  • End user is presented with a permissions request dialog

Environment

  • macOS 10.15 (Catalina)

Solution

  • macOS 10.15 (Catalina) introduced additional security protections which may prevent Take Control connections
  • On first connection, the end user will be presented with a dialog box similar to the below:
  • The end user needs to allow access by clicking the Request Access/Open Full Disk Access Preferences buttons
  • They may need to click the padlock icon in the lower right corner of the Security & Privacy dialog and provide admin credentials
  • It is not possible to enable these access permissions remotely
