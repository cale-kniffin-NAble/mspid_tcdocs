Remote mouse and keyboard not working on Take Control

Last Modified

Tue 12 Jan 10:45 GMT 2021

Description

  • Mouse and keyboard input does not appear on remote end during a session - but local inputs from the end user's mouse and keyboard go through without any issues.

Environment

  • /
  • Take Control

Solution

  • This is a known incompatibility issue with mouse and keyboard sharing applications installed on the remote end - in particular the application Synergy has been found to cause this issue when installed.
  • Current workarounds are to, on the remote end:
    • Uninstall the mouse and keyboard sharing application.
    • Disable services associated to the mouse and keyboard sharing application.
    • Disabling the End User's mouse and keyboard may resolve the issue as follows, during the remote session:
      1. Select Remote Desktop;
      2. Select COMMANDS;
      3. Select Block Remote Keyboard and Mouse.
