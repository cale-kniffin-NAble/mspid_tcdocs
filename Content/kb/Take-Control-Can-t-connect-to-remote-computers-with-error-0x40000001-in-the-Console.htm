Take Control: Can't connect to remote computers with error 0x40000001 in the Console

Last Modified

Tue Mar 19 18:56 GMT 2019

Description

  • Attempting to connect to a device from the console, results in an error:
    • Error: 0x40000001.
  • Logs folder only contains the BAConsoleApp_yyyymmdd_n.log file(s);
  • Trying to run the BASEClient.exe (Viewer) file directly, results in the Windows error message: "The application was unable to start correctly (0xc0000018). Click OK to close the application.";
  • Reinstalling the Console doesn't solve the issue.

Environment

  • Any supported Windows version
  • , version 6.80.22 or later

Solution

  • This issue may be caused due to a damaged console install, which needs to be removed and reinstalled as follows:
    1. Uninstall the existing Take Control Tech Console (can be done via running uninstall.exe under C:\Users\Bruce\AppData\Local\BeAnywhere Support Express\Console );
    2. Reboot the device - this step is vital, following the uninstall;
    3. Go to Take Control: Downloads.
    4. Click Windows Console to download.
    5. Install the downloaded console.
  • Additionally, please review if you have any AV Software with a Behavioral Scan Module that might be automatically flagging Take Control's behavior as suspicious (such as VIPRE) and consider white-listing the Take Control Console folder and it's contents: C:\Users\Bruce\AppData\Local\BeAnywhere Support Express\Console .
