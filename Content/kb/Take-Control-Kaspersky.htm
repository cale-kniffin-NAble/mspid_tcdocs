Take Control: Kaspersky interfering with remote capture

Last Modified

Thu Jul 09 17:28 GMT 2020

Description

  • Kaspersky disables all image capture as a security precaution when interacting with access anti-virus menus
  • What components need to be excluded?

Environment

  • /;
    • .
  • :
    • Take Control.

Solution

  • Some iterations of the Kaspersky AV interfere with Take Control's image capture and remote desktop interaction.
  • The following components will need to be added to the allow list/excluded on the client's device where Kaspersky is installed, located in:
    • Standalone: C:\Program Files\BeAnywhere Support Express\GetSupportService or C:\Program Files (x86)\BeAnywhere Support Express\GetSupportService:
      • BASupSrvc.exe;
      • BASupConHelper.exe;
      • BASupTSHelper.exe.
    • RMM: C:\Program Files\BeAnywhere Support Express\GetSupportService_LOGICNow, C:\Program Files\BeAnywhere Support Express\Take Control Agent or C:\Program Files (x86)\BeAnywhere Support Express\GetSupportService_LOGICNow, C:\Program Files (x86)\BeAnywhere Support Express\Take Control Agent:
      • BASupSrvc.exe;
      • BASupConHelper.exe;
      • BASupTSHelper.exe.
    • : C:\Program Files\BeAnywhere Support Express\GetSupportService_N-Central or C:\Program Files (x86)\BeAnywhere Support Express\GetSupportService_N-Central:
      • BASupSrvc.exe;
      • BASupConHelper.exe;
      • BASupTSHelper.exe.
