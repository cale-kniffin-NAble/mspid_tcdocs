Take Control: Cursor Misaligned between local and remote end

Last Modified

Thu May 16 16:07 GMT 2019

Description

  • During a Take Control session, the cursor's actual location doesn't match the remote cursor's location in some points of the screen.

Environment

  • /;
    • .
  • :
    • Take Control.

Solution

  • Take Control will downscale or upscale when there's a significant resolution or layout difference and this can result in black bars/empty space where your mouse will track but not match the remote cursor location.
  • If the remote resolution does not actually match the native display resolution this can cause issues for the desktop capture and result in misaligned cursors. Please consider changing the Remote Resolution to match the native resolution scale.
    • Note that you can change Remote Resolution via the Take Control session as follows:
      1. Left-click on Remote Desktop to open the drop-down menu;
      2. Hover over View;
      3. Hover over Change Remote Resolution;
      4. Left-click on a resolution that matches the remote system's native display resolution.
