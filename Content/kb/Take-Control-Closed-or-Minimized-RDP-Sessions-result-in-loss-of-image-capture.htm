Take Control: Closed or Minimized RDP Sessions result in loss of image capture

Last Modified

Mon Nov 12 17:22 GMT 2018

Description

  • After closing/minimizing an RDP session to a device, the Take Control session to that device shows nothing in Remote Desktop.

Environment

  • /

Solution

  • During a remote session to a device, Take Control will capture the visible remote desktop - should an ongoing RDP session to that device be terminated/minimized, the Remote Desktop tab will cease image capture (as there's no longer a display to capture);
  • The RDP session will need to be maximized or restarted in order for Take Control to be able to capture the remote display.
