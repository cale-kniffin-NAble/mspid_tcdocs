Take Control: Unable to use Remote Control Mode/Showing page unreachable/rejected for sso.navigatorlogin.com

Last Modified

Thu Aug 08 12:01 GMT 2019

Description

  • When attempting to enable Remote Control mode in the Admin Area, the Remote Control mode widget displays a broken hyperlink with one of the following messages:
    • 404 Page not found;
    • sso.navigatorlogin.com is unreachable;
    • Connection rejected.
  • Browser indicates cookies are blocked while on the Admin Area.

Environment

  • /:
    • Admin Area Remote Control mode.
  • Google Chrome.

Solution

  • The Remote Control widget needs to make use of third-party cookies associated with the sso.navigatorlogin.com domain (used for );
  • In chrome, if Third Party Cookies are blocked, this will result in the widget not being able to work as intended and display connectivity errors. Please consider allowing Third Party Cookies as follows:
    1. On your computer, open Chrome;
    2. At the top right-corner, click on More (vertical ellipsis) and select Settings;
    3. At the bottom of the left pane, click on Advanced;
    4. Under Privacy and security, click on Site settings > Cookies;
    5. Turn Allow sites to save and read cookie data on;
    6. Turn Block Third-Party Cookies off;
    7. Reload the Admin Area and make use of the Remote Control feature as intended.
      • Note that a cache clear and reload is recommended.
  • Note that third-party browser extensions or add-ons can also cause the observed issues, particularly those related to script-blocking, ad-blocking or redirect prevention. Please consider disabling them or white-listing/excluding the Admin Area from any filters.
