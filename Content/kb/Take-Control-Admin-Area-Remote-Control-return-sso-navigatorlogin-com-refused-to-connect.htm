Take Control: Admin Area's Remote Control mode returning "sso.navigatorlogin.com refused to connect" message

Last Modified

Sun Mar 22 23:26 GMT 2020

Description

  • When attempting to click on the Remote Control option via the Admin Area, a blank square is loaded and an error message is received:
    • Error: sso.navigatorlogin.com refused connection!

Environment

  • /:
    • Admin Area's Remote Control mode.

Solution

  • Happens with a technician who's Single Sign-On Status is Enrolled:
    • The browser's settings, add-ons or security software are likely blocking cookies/traffic. Please ensure that web traffic from the following domains is allowed:
      • sso.navigatorlogin.com
      • widget.swi-rc.com
      • admin.swi-tc.com
  • Happens regardless of browser and browser settings in use and the Technician's Single Sign-On Status is Uninvited/Invited:
    • This issue occurs due to the Technician attempting to use Remote Control mode not having completed their SSO Sign-up via Take Control.
    • Re-send the Take Control invitation for the technician to complete their SSO Sign-up. For more information, see [%=General.ProductName_tc-sa%]: SSO Email Invitation/How to resend the SSO Email Invitation.
