How to generate a process dump using ProcDump.exe

Last Modified

Wed Dec 18 10:20 GMT 2019

Description

  • How to generate a process dump using ProcDump.exe
  • Application Event logs show an application crashing due to an issue with a particular dll file:
    • - <System> <Provider Name="Application Error" /> <EventID Qualifiers="0">1000</EventID> <Computer>PC01.local</Computer> </System> - <EventData> <Data>BASupSrvcCnfg.exe</Data> <Data>6.80.21.941</Data> <Data>CFGMGR32.dll</Data> <Data>C:\Program Files (x86)\BeAnywhere Support Express\GetSupportService_N-Central\BASupSrvcCnfg.exe</Data> <Data>C:\Windows\syswow64\CFGMGR32.dll</Data> </EventData>

Environment

Solution

  1. Download the ProcDump.exe to the device that is experiencing the issue:
    • ProcDump.exe
    • NOTE: If a dump is generated from Task Manager it will be a 64-bit process dump, but a 32-bit dump is generally required.
    • If you are uncertain if a 64-bit process dump is required please ask the support technician you are working with.
  2. Extract to a suitable location
  3. Open Command Prompt as administrator
  4. Enter in the following:
    1. cd <location from step 2>
      • For example:cd C:\Users\Administrator\Downloads
    2. procdump.exe -accepteula -ma -t <executable name>
      • For example: procdump -c 80 -u -mp -n 1 -o -w agent.exe
  5. Reproduce the issue
  6. Attach the process dump to the case
  • For more information on ProcDump and its switches, please go here or here
