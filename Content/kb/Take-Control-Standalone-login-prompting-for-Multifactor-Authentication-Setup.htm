login prompting for Multifactor Authentication Setup

Last Modified

Wed Jul 01 14:39 GMT 2020

Description

  • Attempting to login to and a Prompt for Multifactor authentication pops up.
  • The prompt happens at every login attempt.

Environment

  • /

Solution

  • will make MFA a mandatory login step as of Monday, 15th June 2020 for the Administrator Template (Admins) permission and the Main Admin account. What this means is that a username / password combination alone will no longer be sufficient to log in and use the product.
  • This is an SSO level setting so all other products that you sign in using SSO will also have MFA enabled once this has been turned on
  • All other accounts will be presented with a pop up strongly recommending they enable MFA but be allowed to continue without it
  • Notice the new prompt at every login point encouraging everyone to enable MFA
  1. The first step is to download a suitable authenticator app.
  2. By selecting Next, users can finalize the setup process by scanning the QR code, or entering the secure code provided in the authentication app.
  3. Notice the recovery code that should be copied and safely store should a technician need to recover the app.


Once MFA is enabled, all login attempts for your account will only be successful with the input of your Username, Password and Authenticator code. MFA adds a powerful layer of security that will help keep your account, and your clients, safe from password attacks. For more information, see:
