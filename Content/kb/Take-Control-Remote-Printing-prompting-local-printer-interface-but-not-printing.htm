Take Control Remote Printing prompting local printer interface but not printing

Last Modified

Wed Mar 18 15:13 GMT 2020

Description

  • With Remote Printing enabled with Take Control/, selecting to print a remote file to the Remote Printer and then approving the print on the local interface causes nothing to occur.
  • Transferring the same file through file transfer and printing it locally works without any issues.
  • Remote Printing with Print to PDF or other printers works without any issues.
  • Viewer Logs Display:
    • 2018-05-01 10:59:24:512 \{9932} [PrintXPSFile] - File - "<FILE LOCATION>" | Printer - "<INCOMPLETE PRINTER NAME>" | printJobName - " Remote Control Viewer Print job"

Environment

  • /
    • Take Control

Solution

  • This issue can be caused by the printer's network name, in particular, if the name is too large, the will not be able to parse the name completely and, since the truncated name's printer does not exist in the network, it will not be able to print - this can be validated in the Viewer logs if the Printer name shows up incomplete in the PrintXPSFile action.
  • Consider editing the printer's network name to one equal to or under 30 characters long.
