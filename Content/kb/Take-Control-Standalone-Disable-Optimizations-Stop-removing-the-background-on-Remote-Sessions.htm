: Disable Optimizations/Stop removing the background on Remote Sessions

Last Modified

Tue May 07 16:28 GMT 2019

Description

  • The Take Control Agent keeps removing the remote desktop background on connections, leaving it black/blue;
  • How to disable the Agent's removal of clear type fonts, background and aero effects.

Environment

  • /

Solution

  • For Applets:
    1. Login to your Admin Area;
    2. Navigate to Profile > Applet Setup
    3. Under the Settings tab, un-mark the option: Remove the remote desktop background automatically
    4. Save the new settings ?
      • This will only apply to newly generated Applets, Applets generated prior to these settings being in place will not use the new settings
  • For Agents, you will need to alter the settings on the installed Agent itself:?
    1. Connect to the remote device
    2. Restore/Open the Agent Configuration Window (Should be located in the systray or by searching for "Take Control Agent Configuration")
      • Be sure it's open in Admin Mode (if the lock on the top right is closed, click it to unlock Admin Mode)
    3. Open the SETTINGS menu
    4. Make sure the following option isn't marked: Optimize Desktop Settings when in session
    5. APPLY the new settings

Article Number: 110077

Internal notes: Due to conflicts with permissions, Windows Aero settings or unexpected session interruptions - the background may sometimes not be restored after the session ends, these steps will also assist in correcting this issue
