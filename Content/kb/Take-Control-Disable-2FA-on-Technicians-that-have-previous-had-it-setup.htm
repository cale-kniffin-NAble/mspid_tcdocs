Take Control: Disable 2FA/MFA on Technicians that have previously had it setup

Last Modified

Tue Aug 04 12:40 GMT 2020

Description

  • A Technician has setup 2FA/MFA on their login for Take Control and needs to revoke it, rather than reset it.

Environment

  • /.

Solution

  • To remove or reset 2FA from the SSO login of a Take Control Technician, the Technician themself needs to perform the following steps:
    1. Login to the Admin Area.
      • The Technician does not require Admin Area Access permissions to complete the following steps.
    2. Select the User Menu on the top right corner.
    3. Select Two-Factor Authentication.
    4. On the SSO 2FA Page, you'll be request to input either your Recovery Code or the current 6-digit key generated by your Authenticator app, please do so and select Disable 2FA. 
      • If you no longer have access to your Recovery Code or Authenticator app, ask your Main Admin to perform the steps listed in [%=General.ProductName_tc-sa%]: Lost recovery code and need to reset or revoke 2FA.
    5. You'll receive confirmation that 2FA has been disabled for this login on and you should now be able to login to the Take Control Admin Area, Desktop Console or Mobile Console and be presented with a pop-up message that allows you to either setup 2FA/MFA again or continue without a new setup.
  • In order to remove or reset 2FA from the SSO login of an End User, the End User themself needs to perform the following steps:
    1. Login to End User area.
    2. Select the User Menu on the top right corner.
    3. Select Two-Factor Authentication.
    4. On the SSO 2FA Page, you'll be request to input either your Recovery Code or the current 6-digit key generated by your Authenticator app, please do so and select Disable 2FA;
      • If you no longer have access to your Recovery Code or Authenticator app, ask your Support Provider's Main Admin to perform the steps listed in [%=General.ProductName_tc-sa%]: Lost recovery code and need to reset or revoke 2FA.
    5. You'll receive confirmation that 2FA has been disabled for this login on and you should now be able to login to the End User area and be presented with a pop-up message that allows you to either setup 2FA/MFA again or continue without a new setup.
  • NOTE: We heavily recommend that all users, including End Users and Technicians, make use of 2FA/MFA and will be prompted to do so each time they login, although they will have the option to continue without completing setup. Profiles set with Administrator permissions and that of the Main Admin will not be allowed to login without completing 2FA/MFA setup.
