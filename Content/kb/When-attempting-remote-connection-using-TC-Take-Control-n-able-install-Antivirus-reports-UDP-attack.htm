When attempting to perform a remote connection using Take Control, the install Antivirus reports it as as 'UDP Port scanning attack'

Last Modified

Tue 12 Jan 16:38 GMT 2021

Description

  • When attempting to perform a remote connection using Take Control, the install Antivirus reports it as as 'UDP Port scanning attack'

Environment

  • /
  • Take Control

Solution

  • The most likely cause of this is the feature "Attempt Peer-To-Peer" is enabled.
  • Disable this option within the profile so it no longer alerts on this.
  • For more information on this for Take Control Standalone, see [%=General.ProductName_tc-sa%]: Ports and Domains/Firewall and AV Exclusions.
  • For more information on this for , see Remote Control settings.
  • For more information on this for , see Take Control/ Firewall Ports and Domain (Integrations in and )
