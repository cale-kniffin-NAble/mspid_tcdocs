Take Control: Blank/Blue Screen when connecting to a device with no text

Last Modified

Tue Jan 05 12:31 GMT 2021

Description

  • Connecting to a remote device shows a blank or blue screen with no message, CTRL+ALT+DEL does nothing;
  • System Info tab reveals an existing Graphics adapter and a standard resolution (4:3).

Environment

  • /;
    • .
  • :
    • Take Control.

Solution

  • Issue is caused by a third party application managing display or power settings - please review if there's third party software (particularly security software or power management software) that would force a display sleep.
  • Current workaround is force the display to sleep and then wake, refreshing the capture:
    1. Select the Remote Desktop tab;
    2. Select the drop-down menu to the right of Remote Desktop (looks like a downward arrow);
    3. Select Commands > Show Blank Screen (hotkeys: Shift+Ctrl+N) to sleep the display;
    4. Select Commands > Show Blank Screen (hotkeys: Shift+Ctrl+N) to wake up the display.
