Configure Port Forwarding

Configure Agents to act as a gateway between local applications and services on the remote LAN that are not accessible from an outside network. This provides nearly the same level of functionality as a Virtual Private Network.

Use to create temporary or permanent port forwards to remote networks for increased performance, security and simplicity in connecting to remote services. For security purposes, this feature is setup per remote machine, and the permanent forwards are saved on the configuration of each console. This means that they need to be recreated if the technician uses another console.

What do you want to do?

  • Configure port forwarding on the Agent (windows only)
  • Configure port forwarding on the Console
  • Configure an FTP connection
