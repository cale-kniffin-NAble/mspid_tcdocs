In Chrome version 77 and later, the checkbox for "always open these types of links in the associated app" is no longer displayed for URL custom protocols. The impact of this is that for all users on the Chrome browser, there is no longer an option to remember your protocol handling preferences, and the below dialog (or some variation, pending your local settings, etc) will be presented each time a session is launched. This is, in no way a bug, or a fault. Further, there is nothing the team can do to mitigate this. It is simply a change about how custom protocols are handled within the Chrome browser.

Access to the network

's list of IP addresses is very large and highly dynamic. Many IP address ranges point towards the intelligent routing infrastructure that forms part of our global network. Additionally, it is not possible to curate a list of Gateway IP addresses – the components responsible for session connectivity. This type of IP address is negotiated upon session setup and is predicated by the geo-location of both the session host, the session recipient, as well as being subject to available network conditions.

To get the best performance out of , please make sure that there is no deep packet inspection applied to the domains. On the Agents' networks, you can also create an exclusion on inspecting outbound traffic on the TCP port 3377.
